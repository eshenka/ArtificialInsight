# Gateway Service

## Overview

This service acts as an HTTP gateway, providing a simple RESTful API interface for users to interact with the underlying RAG (Retrieval-Augmented Generation) system managed by the `controller` gRPC service. It receives user prompts via HTTP, forwards them to the controller service using gRPC, and returns the generated answer back to the user via HTTP.

## Technology Stack

*   **Language:** Python 3
*   **Framework:** FastAPI
*   **Communication Protocol (Internal):** gRPC (communicates with the `controller` service)
*   **Communication Protocol (External):** HTTP/JSON

## API Description

### Answer Prompt

*   **Endpoint:** `/answer`
*   **Method:** `POST`
*   **Description:** Receives a user prompt and forwards it to the controller service to get an answer using the RAG pipeline associated with the provided token.
*   **Headers:**
    *   `Authorization: <user_token>` (Replace `<user_token>` with the actual user authentication token)
*   **Request Body:**
    ```json
    {
        "prompt": "Your question or prompt here"
    }
    ```
*   **Success Response (200 OK):**
    ```json
    {
        "answer": "The answer generated by the RAG system."
    }
    ```
*   **Error Responses:**
    *   `401 Unauthorized`: If the `Authorization` header is missing or the token is invalid.
    *   `422 Unprocessable Entity`: If the request body is malformed (e.g., missing `prompt`).
    *   `500 Internal Server Error`: If there's an issue communicating with the backend gRPC service or an unexpected error occurs.

## Internal Communication

The gateway service communicates with the `controller.Controller` gRPC service defined in `proto/controller.proto`. Specifically, it calls the `AnswerPrompt` RPC method, passing the user token and prompt.
