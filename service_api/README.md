# Gateway Service

## Overview

This service acts as an HTTP gateway, providing a simple RESTful API interface for users to interact with the underlying RAG (Retrieval-Augmented Generation) system managed by the `controller` gRPC service. It receives user prompts via HTTP, forwards them to the controller service using gRPC, and returns the generated answer back to the user via HTTP. It also allows creating new RAG pipelines via a form-based request.

## Technology Stack

*   **Language:** Python 3
*   **Framework:** FastAPI
*   **Communication Protocol (Internal):** gRPC (communicates with the `controller` service)
*   **Communication Protocol (External):** HTTP/JSON, HTTP/Form-Urlencoded

## API Description

### Answer Prompt

*   **Endpoint:** `/answer`
*   **Method:** `POST`
*   **Description:** Receives a user prompt and forwards it to the controller service to get an answer using the RAG pipeline associated with the provided token.
*   **Headers:**
    *   `Authorization: <user_token>` (Replace `<user_token>` with the actual user authentication token)
*   **Request Body (`application/json`):**
    ```json
    {
        "prompt": "Your question or prompt here"
    }
    ```
*   **Success Response (200 OK):**
    ```json
    {
        "answer": "The answer generated by the RAG system."
    }
    ```
*   **Error Responses:**
    *   `401 Unauthorized`: If the `Authorization` header is missing or the token is invalid.
    *   `422 Unprocessable Entity`: If the request body is malformed (e.g., missing `prompt`).
    *   `500 Internal Server Error`: If there's an issue communicating with the backend gRPC service or an unexpected error occurs.
    *   `503 Service Unavailable`: If the backend controller service is unavailable.
    *   `504 Gateway Timeout`: If the request to the backend controller service times out.

### Create Pipeline

*   **Endpoint:** `/pipeline`
*   **Method:** `POST`
*   **Description:** Creates a new RAG pipeline configuration by forwarding the details to the controller service. Accepts data via `application/x-www-form-urlencoded`.
*   **Request Body (`application/x-www-form-urlencoded`):**
    *   `user_name`: (string, required) - Identifier for the user creating the pipeline.
    *   `description`: (string, required) - A description for the new pipeline.
    *   `language`: (string, required) - The primary language for the pipeline (e.g., "en", "es").
    *   `entry_docs_url`: (string, required) - The starting URL for documentation scraping.
    *   `rules`: (string, required) - A **JSON string** representing the scraping rules. The structure must match the `ScrapeRules` message defined in `common.proto`. See example below.

*   **Example `rules` JSON string structure:**
    ```json
    {
      "max_depth": 5,
      "max_pages": 100,
      "scrape_patterns": [
        {
          "url": { "pattern": "https://docs\\.example\\.com/.*" },
          "css_selector": "main .content"
        },
        {
          "url": { "pattern": "https://help\\.example\\.com/articles/.*" }
        }
      ],
      "forbidden_urls": [
        { "pattern": ".*login.*" },
        { "pattern": ".*\/edit\/.*" }
      ]
    }
    ```
    *(Note: When sending this as form data, the JSON string itself needs to be properly URL-encoded by the client.)*

*   **Success Response (200 OK):**
    ```json
    {
        "token": "The unique token generated for the new pipeline."
    }
    ```
*   **Error Responses:**
    *   `422 Unprocessable Entity`: If form data is missing, the `rules` field is not valid JSON, or the structure within the `rules` JSON is invalid according to `ScrapeRules`. Also used if the controller service rejects the configuration as invalid.
    *   `500 Internal Server Error`: If there's an issue communicating with the backend gRPC service or an unexpected error occurs.
    *   `503 Service Unavailable`: If the backend controller service is unavailable.
    *   `504 Gateway Timeout`: If the request to the backend controller service times out.

## Internal Communication

The gateway service communicates with the `controller.Controller` gRPC service defined in `proto/controller.proto`. It calls the `AnswerPrompt` and `CreatePipeline` RPC methods.
